## История разработки

Проект разработан в рамках дипломной работы как демонстрация применения AI-технологий в e-commerce. 

**Основные этапы:**
1. **Прототипирование** — создание базового Flask-приложения с чат-интерфейсом
2. **Интеграция AI** — подключение OpenRouter API для генерации ответов
3. **Оптимизация** — внедрение кэширования, lazy loading изображений, динамического подбора моделей
4. **Локальная база знаний** — создание JSON-базы для мгновенных ответов на частые вопросы
5. **Деплой** — размещение на бесплатном хостинге PythonAnywhere

## Архитектура и паттерны

### Разделение ответственности (Separation of Concerns)

Проект следует принципу единственной ответственности (SRP):

**Backend (`app.py`):**
- Маршрутизация HTTP-запросов (Flask routes)
- Интеграция с OpenRouter API
- Работа с базой знаний (поиск по ключевым словам)
- Динамический подбор бесплатных AI-моделей через API OpenRouter
- Проксирование запросов к Open Library API для получения данных о книгах

**Frontend (`static/script.js`):**
- Управление состоянием приложения (чат, пагинация)
- Асинхронные запросы к backend (fetch API)
- Рендеринг UI (карточки книг, сообщения чата)
- Кэширование страниц книг (booksCache)
- Ленивая загрузка изображений (loading="lazy")

**Стили (`static/style.css`):**
- Минималистичный дизайн в книжной эстетике
- Адаптивная верстка (CSS Grid, Flexbox)
- CSS-переменные для темизации

**Шаблоны (`templates/index.html`):**
- Семантическая HTML-разметка
- Интеграция статических ресурсов
- Мета-теги для SEO и предзагрузки соединений

### Паттерны проектирования

1. **Model-View-Controller (MVC)** — разделение данных (модели книг), представления (HTML/CSS) и контроллера (Flask routes)
2. **Repository Pattern** — абстракция доступа к данным (Open Library API, база знаний)
3. **Chain of Responsibility** — каскадная обработка запросов: сначала поиск в локальной базе, затем перебор бесплатных AI-моделей
4. **Factory Pattern** — динамическое создание виртуальных консультантов через системные промпты

## Сторонние ресурсы

### 1. Open Library API
**Назначение:** Подгрузка карточек книг с обложками, авторами, годами издания.

**Endpoint:** `https://openlibrary.org/search.json`

**Параметры:**
- `q` — поисковый запрос (жанр, название)
- `page` — номер страницы пагинации
- `limit` — количество книг (12)
- `fields` — поля: title, author_name, cover_i, first_publish_year

**Особенности:**
- Бесплатный доступ без API-ключа
- Возможность пагинации (до 1000 результатов)
- Обложки через `https://covers.openlibrary.org/b/id/{cover_i}-M.jpg`

### 2. OpenRouter API
**Назначение:** Доступ к бесплатным AI-моделям (LLM) для генерации ответов.

**Endpoint:** `https://openrouter.ai/api/v1/chat/completions`

**Механизм работы:**
1. Запрос к `/api/v1/models` для получения актуального списка бесплатных моделей
2. Фильтрация моделей с `pricing: "0"` или суффиксом `:free`
3. Кэширование списка на 1 час
4. Последовательный перебор моделей до успешного ответа
5. Обработка ошибок: 429 (Rate Limit), 404 (Model Not Found)

**Используемые модели (примеры):**
- meta-llama/llama-3.1-8b-instruct:free
- qwen/qwen-2.5-7b-instruct:free
- google/gemma-3-1b-it:free

### 3. Преимущества подхода
✅ **Полностью бесплатная инфраструктура**  
✅ **Отказоустойчивость** — автоматический переход к следующей модели при ошибке  
✅ **Актуальность** — динамическое получение списка доступных моделей  
⚠️ **Возможны задержки** — из-за использования бесплатных ресурсов (Open Library и бесплатные модели OpenRouter)
